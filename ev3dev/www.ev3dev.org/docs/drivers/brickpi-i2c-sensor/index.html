    A brickpi-i2c-sensor device is loaded by the brickpi driver when manually specified by setting the brickpi-in-port to nxt-i2c mode and writing the device name and address to set_device. You can use any one of the sensors that has the nxt-i2c-sensor module from the list of supported sensors. Not all functionality of a sensor may be supported when connected to a brickpi-in-port. For these, you should use input port 5 on the BrickPi instead.

    sysfs attributes

    Devices bound to this driver can be found in the directory /sys/bus/lego/drivers/brickpi-i2c-sensor/. However, these sensors use the lego-sensor class which is where the useful stuff is. Follow the link for more information.
