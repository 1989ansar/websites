    Note: If you do not need to access the Internet from the EV3 brick, consider using tethering instead.

    These instructions are for brickman v0.5.0. If you are using an older version, please upgrade.

    • Note: These instructions were written using OSX version 10.9, but should work for other versions as well.

      1. On the EV3, first verify that the CDC driver is enabled. In brickman, open the Wireless and Networks menu and select USB. Make sure CDC is selected and active.

      2. On your Mac, open up System Preferences and select Network.

      3. Once you get the Network dialog, click on the + icon in the lower left area to add a new network device. You’ll need to select the CDC Composite Gadget in the drop down box. The name will be similar to what is shown below. I have renamed the service “ev3dev” so it’s easier to keep track of later. Click Create when you are done.

      4. Click Apply to save your changes. After a short time, the ev3dev entry (or whatever you named it) should show connected and have a Self-Assigned IP address

      5. If you don’t need to access the Internet from your EV3 over this connection, manually configure the IP address to 192.168.2.1. Note: you won’t be able to update packages without an Internet connection.

        Then skip to step 7.

        If you do want to be able to connect to the Internet from the EV3, continue with the next step.

      6. To share our Internet connection with the EV3. Go back to System Preferences and select Sharing.

        Click Internet Connection on the left, but don’t check the box yet. On the right, Share your connection from: will be Wi-Fi (or Ethernet if you have a wired connection). Also check the box next to CDC Composite Gadget

        Now check the box next to Internet Connection on the left to enable it. Read the warning and then click Start.

      7. Now, we need to assign an IP address to our EV3. In brickman, go to Networking and select Manage connections…, then select the Wired connection.

        On the IPv4 tab, select Change….

        Then choose Load Mac Defaults.

        The settings on the IPv4 tab will remain blank because we are not connected yet. On the Conn. tab, check the box for Connect automatically. This way you don’t have to connect manually after you reboot.

        Then select Connect. State should change to Online. Go back to the IPv4 tab. It will show the IP address of your EV3.

      8. For ssh access to the EV3 under OS X, you can use the good old ssh program from the terminal window. I’m sure there are other solutions, and if you send me your suggestions I’ll add them to a list.

        In a terminal, run the following command.

        host:~ user$ ssh root@192.168.2.3

        If you have never connected before, you will prompted to confirm the authenticity of the host, so type yes when prompted.

        The authenticity of host '192.168.2.3 (192.168.2.3)' can't be established.
RSA key fingerprint is xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx.
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added '192.168.2.3' (RSA) to the list of known hosts.

        The default root password is r00tme.

    • Note: This instructions are written for Ubuntu 14.04 using the default desktop (aka Unity).

      1. On the EV3, first verify that the CDC driver is enabled. In brickman, open the Wireless and Networks menu and select USB. Make sure CDC is selected and active.

      2. On your host computer, with the EV3 connected via USB, open up System Settings and select Network

      3. In the Network dialog, you should see a Wired connection that says Connecting. It may say Disconnected instead if it has timed out already. You can also idetentify the connection by the Hardware Address - it will start with 12:16:53. Click the Options… button for this connection.

      4. In the dialog that open, change the Connection Name to whatever you want (using ev3dev here). Select the IPv4 Settings tab and change the Method to Shared to other computers.

        Click Save… when you are done. The connection will now say that it is connected.

      5. Now, we need to assign an IP address to our EV3. In brickman, go to Wireless and Networks and select All Network Connections, then select the Wired connection with the USB icon.

        On the IPv4 tab, select Change….

        Then choose Load Linux Defaults.

        The settings on the IPv4 tab will remain blank because we are not connected yet. On the Conn. tab, check the box for Connect automatically. This way you don’t have to connect manually after you reboot.

        Then select Connect. State should change to Online. Go back to the IPv4 tab. It will show the IP address of your EV3.

      6. Now we are going to connect to the EV3 using ssh.

        user@host:~$ ssh root@10.42.0.3

        If you have never connected before, you will prompted to confirm the authenticity of the host, so type yes when prompted.

    • Note: These instructions were written using Windows 7, but they will work for other versions too.

      1. On the EV3, first verify that the RNDIS driver is enabled. In brickman, open the Wireless and Networks menu and select USB. Make sure RNDIS is selected and active.

      2. Connect your EV3 to your Windows computer using the USB cable that came with the EV3.

      3. When it is connected, Windows will automatically install a driver for it. It installs the wrong driver! We need to fix this. Open up Control Panel > Hardware and Sound and select Device Manager

      4. In Device Manager, find your EV3’s RNDIS device. It is under Network Adapters > USB Ethernet/RNDIS Gadget.

      5. Double-click it and select the Drivers tab. It says that it is using an Acer driver, which doesn’t work properly with ev3dev. Click Update driver… to change the driver.

      6. Select Browse my computer for driver software.

      7. Then Let me pick from a list of device drivers on my computer.

      8. Uncheck the box that says Show compatible hardware. In the Manufacturer list, pick Microsoft Corporation (in Windows 8, pick Microsoft). In the Network Adapter list, pick Remote NDIS Compatible Device. Then click Next.

      9. Windows thinks we don’t know what we are doing, but really we do, so click Yes.

      10. When it is done installing, click Close.

      11. Yea! We have the right driver now! Click Close again. You can close the Device Manager too. We are done with it.

      12. Now, open up the Network and Sharing Center either using the notification tray icon or using Control Panel > Network and Internet > Network and Sharing Center. Click on Change adapter settings.

      13. There will be a connection that says USB Ethernet/RNDIS Gadget.

      14. This is the EV3. Let’s rename it so it is easy to identify. Right-click the USB Ethernet/RNDIS adapter and select Rename. Call it EV3 or whatever you like.

      15. If you don’t need to access the Internet from your EV3 over this connection, manually configure the IP address to 192.168.137.1. Note: you won’t be able to update packages without an Internet connection.

        Then skip to step 18.

        If you do want to be able to connect to the Internet from the EV3, continue with the next step.

      16. To share our Internet connection with the EV3, Double-click the other network connection (not the one we just renamed). This will show the connection status. If you have more than one other connection, you will want to verify that this connection is indeed the Internet connection.

      17. Click on the Properties button. In the window that opens, select the Sharing tab and check the box that says Allow other network users to connect through this computer’s Internet connection.

        If you have more than two network connections, it will ask for a “Home networking connection”. If you see this, select your EV3’s network adapter from the list (this will be called EV3 if you renamed it earlier in the guide).

        Click OK when you are done and close the status window as well. Windows will automatically reconfigure your other network connections.

      18. Now, we need to assign an IP address to our EV3. In brickman, go to Networking and select Manage connections…, then select the Wired connection.

        On the IPv4 tab, select Change….

        Then choose Load Windows Defaults.

        The settings on the IPv4 tab will remain blank because we are not connected yet. On the Conn. tab, check the box for Connect automatically. This way you don’t have to connect manually after you reboot.

        Then select Connect. State should change to Online. Go back to the IPv4 tab. It will show the IP address of your EV3.

      19. Now we are going to connect to the EV3 using ssh. To do this, you need a program called PuTTY.

        Start PuTTY. In the PuTTY Configuration window, type the IP address that we just saw (192.168.137.3). Then click the Open button to connect.

        The first time you connect, you’ll get a warning about the new RSA fingerprint.

        Once you’re connected, type in the userid as root and the password as r00tme and you should be connected.
